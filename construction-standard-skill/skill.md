# CONSTRUCTION STANDARD SKILL

## 一、模式定位

本 Skill 仅用于生成“施工方案”。

目标文档性质：
- 指导施工队实际施工
- 明确施工步骤与工艺要求
- 明确安全与文明施工要求
- 明确验收结构

禁止生成：
- 报价内容
- 投标性能参数
- 厂商宣传语言
- 大量设备性能介绍

---

## 二、输入要求

模型必须基于以下两类输入生成施工方案：

① 设备清单  
② 施工补充说明（如有）
③ 施工组织要素（如提供）
若无施工补充说明，则默认按标准弱电施工规范执行。

---

## 三、自动章节生成规则

模型必须根据设备清单自动识别系统类型，并生成对应施工章节。
若提供施工组织要素，必须在“施工组织安排”章节中结构化引用。

### 1️⃣ 通用基础章节（必须生成）

一、项目概述  
二、施工依据  
三、施工组织安排  
四、施工阶段划分  
五、安全施工措施  
六、文明施工措施  
七、项目风险及控制  

---

### 2️⃣ 根据设备自动生成分系统章节

如果清单包含：

- 摄像机 → 生成《视频监控系统施工》
- 门禁设备 → 生成《门禁系统施工》
- RFID设备 → 生成《RFID系统施工》
- 交换机/机柜 → 生成《机房及网络设备安装》
- 数据采集设备 → 生成《终端设备安装》
- 综合布线材料 → 生成《综合布线施工》

每个系统施工章节必须包含：

1. 施工准备  
2. 布线规范  
3. 设备安装要求  
4. 接线要求  
5. 调试流程  
6. 施工注意事项  

---

## 四、施工内容表达规则

【总体压缩规则】

1. 《项目概述》不得复述设备清单内容。
2. 《施工依据》仅列依据名称，不得展开解释。
3. 不得重复施工补充说明全文。
4. 所有章节禁止重复背景性说明。
5. 禁止生成总结性套话段落。
6. 禁止出现“本项目旨在”“本系统通过”“为保障”“进一步提升”等概述性表达。

【施工表达规则】

1. 施工内容必须以“操作步骤”形式表达。
2. 每条描述必须为可执行动作。
3. 每条描述长度控制在一句话以内。
4. 不得重复设备技术参数。
5. 不得重复设备数量说明。
6. 仅引用设备名称。
7. 不得复述设备清单内容。
8. 不得解释系统功能原理。

示例表达方式：

✔ 安装高度范围  
✔ 线缆规格  
✔ 走线方式  
✔ 强弱电隔离要求  
✔ 固定方式  
✔ 调试顺序  

禁止表达：

✘ 产品优势  
✘ 性能指标宣传  
✘ 兼容性吹嘘  
✘ 背景说明段落  
✘ 原理性说明

## 五、安全施工规则

本章节不得由模型自由生成。

必须直接引用文件：

Safety_and_Civil.md

调用方式：
- 完整插入《安全施工措施》章节内容
- 不得删减
- 不得改写
- 不得重组结构

若文件不存在，必须中止生成并提示：
“缺少 Safety_and_Civil.md 文件”

---

必须直接引用文件：

Safety_and_Civil.md

调用方式：
- 完整插入《文明施工措施》章节内容
- 不得删减
- 不得改写
- 不得重组结构

若文件不存在，必须中止生成并提示：

## 七、验收结构生成规则

必须生成验收结构表，包含：

1. 设备数量核对
2. 安装位置核对
3. 供电与通讯测试
4. 功能测试
5. 系统联调测试
6. 施工资料交付

验收结果必须采用：

□ 合格  
□ 不合格  

格式。

---

## 八、图纸说明规则

若未提供CAD图纸：
仅生成图纸目录清单（固定四项，不得扩展说明）：

- 平面布点图
- 线路走向图
- 设备接线图
- 机柜布置图

图纸不生成CAD文件，仅生成说明目录。

---

## 九、输出结构固定格式

输出顺序必须为：

1、施工组织设计正文  
2、分系统施工章节  
3、安全施工措施  
4、文明施工措施  
5、验收结构  

禁止打乱顺序。

---

## 十、语言风格

必须：

- 工程化
- 简洁
- 可执行
- 不使用营销语言
- 不写口号

文档应适合打印归档。


## 十一、图件输出规则
当提供 layout_parameters 时：

1. 前置约束（输入规范）

layout_parameters 必须符合《通用空间布点参数规范.md》。

若不符合，模型必须输出纯 JSON错误对象，并终止图件 JSON 生成：

{
  "error": "layout_parameters 不符合《通用空间布点参数规范.md》"
}

不得输出其他内容。

2. 输出规则（JSON 强制）

模型必须：

按《图件结构化 JSON 输出规范.md》输出标准 JSON。

输出必须为纯 JSON。

输出必须包含以下顶层字段：

layout_data

validation

optimization_suggestions

validation 字段必须存在。

optimization_suggestions 不超过 3 条。

3. 禁止项（严格）

模型禁止：

生成 SVG / HTML

输出坐标计算

输出样式代码

输出 JSON 以外内容

4. 渲染职责

图件由 Python 渲染程序生成。
