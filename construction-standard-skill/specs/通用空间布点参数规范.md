《通用空间布点参数规范 V1.0》

（原 layout_parameters 规范的工程通用版）

一、规范目的

本规范用于统一“任何空间类型”的布点参数输入格式。

适用于：

平面布点图

设备安装图

视频监控布点

弱电平面布置

机房布置

综合空间设备布点

目标：

输入结构统一

模型解析稳定

JSON 输出稳定

Python 渲染长期稳定

二、总体原则

所有空间必须参数化

所有设备必须结构化

禁止模糊表达

禁止口语说明

所有尺寸必须带单位

三、标准输入结构

必须采用如下通用结构：

版本：1.0
图件类型：空间平面布点

空间信息：
  空间数量：1
  空间名称：询问室
  尺寸：
    长度（m）：6
    宽度（m）：4
    层高（m）：3

固定设施：
  - 名称：工作台
    宽度（m）：2
    深度（m）：0.8
    靠墙方向：北
    距墙距离（m）：0.3

设备清单：
  - 类型：摄像机
    数量：2
    安装方式：墙角高位
    距墙距离（mm）：300
    安装高度（m）：2.8
    布置策略：对角覆盖

  - 类型：拾音器
    数量：1
    安装位置：工作台中心
四、字段设计逻辑（通用化）
1️⃣ 空间信息（必填）

空间数量

空间名称

尺寸（长宽高）

2️⃣ 固定设施（可选）

用于：

桌子

柜体

立柱

隔断

设备台

每个设施必须包含：

名称

尺寸

位置规则

3️⃣ 设备清单（必填）

每个设备必须包含：

类型

数量

安装方式

安装位置规则

可选：

布置策略

安装高度

距墙距离

五、允许的布置策略（枚举）

布置策略必须为枚举值，不允许自由描述：

对角覆盖

四角覆盖

单侧覆盖

中心辐射

周界覆盖

均匀分布

六、禁止内容

不得出现：

施工说明

产品型号

技术参数宣传

原理说明

施工步骤

自由描述性语句

本文件仅为：

空间布点逻辑输入文件

七、模型处理规则

当提供符合本规范的参数时：

模型必须：

解析全部字段

输出结构化 JSON（固定结构）

输出布点合理性校验结果

输出不超过3条优化建议（每条≤25字）

模型禁止：

输出 SVG / HTML

输出坐标

输出样式代码

输出自然语言段落说明

图件由 Python 渲染程序生成。

八、为什么这个版本是“通用的”

它不再包含：

接报案室

报案台

民警

当事人

所有功能性角色被抽象为：

固定设施

设备类型

安装规则

布置策略

这叫：

领域抽象

九、长期稳定性结论

只要输入符合本规范：

模型解析稳定

JSON 输出稳定

渲染长期可运行

可扩展到任何空间类型
