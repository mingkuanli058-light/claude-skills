# 图件注册与生成规则（Chart Registry）V3 --- 低 Token 图件系统

版本目标： - 将图件生成 Token 降低 80%--95% - AI
只生成结构数据（JSON） - HTML 由模板渲染 - 图件生成稳定、可重复
图件视觉表达必须遵循：

specs/chart_style.md

------------------------------------------------------------------------

# 一、文件定位

本文件定义 **图件生成标准与规则**。

本文件只负责：

-   图件类型定义
-   图件数据来源规则
-   图件校验规则
-   图件风格控制
-   JSON 图件结构规范

本文件 **不决定是否生成图件**。

是否生成图件由：

specs/方案需要生成图件确认.md

决定。

------------------------------------------------------------------------

# 二、图件生成优先规则

在生成任何图件之前必须执行：

步骤1\
读取：

specs/方案需要生成图件确认.md

步骤2\
确认需要生成的图件列表

步骤3\
根据本文件生成 **chart_spec.json**

步骤4\
由模板系统生成 HTML

步骤5\
HTML 导出 PNG

AI **不直接生成 HTML 图件代码**。

------------------------------------------------------------------------

# 三、图件生成架构

V3 图件系统架构：

需求 / 投资模型 ↓ chart_spec.json ↓ HTML模板渲染 ↓ PNG输出

AI只负责：

生成 chart_spec.json

HTML由模板生成。

------------------------------------------------------------------------

# 四、chart_spec.json 结构

所有图件必须使用统一 JSON 结构。

示例：

    {
      "chart_type": "system_architecture",
      "title": "系统总体架构",
      "nodes": [
        {"id": "client", "type": "terminal", "label": "客户端"},
        {"id": "app", "type": "server", "label": "应用服务器"},
        {"id": "storage", "type": "storage", "label": "存储阵列"}
      ],
      "links": [
        {"from": "client", "to": "app"},
        {"from": "app", "to": "storage"}
      ]
    }

AI只输出结构数据。

------------------------------------------------------------------------

## A-01 系统总体架构图（应用软件）

chart_type

system_architecture

数据来源

specs/需求.md
specs/investment_schema.md
output/建设方案.md

规则
本图采用固定架构结构。
六层结构必须完整出现，不得新增或删除层级。

六层：

用户层  
业务应用层  
业务支撑层  
数据层  
网络层  
硬件层  

三体系（pillars）：

标准规范体系
信息安全体系
运行管理体系

三体系生成规则：
- system_architecture 图件必须包含 `pillars` 字段
- 固定值：`["标准规范体系", "信息安全体系", "运行管理体系"]`
- 渲染为右侧纵向贯穿体系栏，自动适配层区域宽度
- 禁止修改顺序或名称，禁止添加/删除条目

仅表达系统逻辑架构。

节点必须来源于数据文件，不得由 AI 推理新增。

节点必须按语义归属对应层级，不得跨层放置。

允许节点类型：

用户
系统
平台
模块
服务
数据库
设备类型

设备类型仅允许抽象类别：

存储设备
网络设备
监控设备
门禁设备
报警设备
显示设备
安全设备
供电设备

禁止出现具体设备：

服务器
交换机
摄像机
存储阵列
路由器
防火墙

输出

output/chart/system_architecture.html

# 十一、图件生成纪律

所有图件必须：

1 先生成 JSON 结构\
2 使用 HTML 模板渲染\
3 HTML 导出 PNG\
4 分辨率 ≥ 1920×1080

AI禁止：

直接生成 HTML 图件代码

4 chart_registry\
