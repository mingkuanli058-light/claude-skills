# 方案图件生成确认清单

本文件用于确认本项目建设方案需要生成的技术图件类型。

本清单属于 **方案表达控制文件**，  
用于控制方案中图件生成范围，避免模型自动生成不必要图件。

图件生成规则必须服从：

- specs/chart_registry.md
- specs/需求.md
- specs/investment_schema.md
- output/建设方案.md

---

# 五、图件生成控制规则

1. 本文件为 **图件生成唯一确认依据**。

2. 仅允许生成状态为 **必须生成** 或 **条件生成** 的图件。

3. 图件生成规则必须服从：

```
specs/chart_registry.md
```

4. 图件数据来源必须服从以下文件：

* `specs/需求.md`
  → 提供 **现网设备结构**

* `specs/investment_schema.md`
  → 提供 **建设设备清单**

* `output/建设方案.md`
  → 提供 **系统结构、模块划分、数据流向、部署逻辑**

5. 图件中不得新增未在以下文件中定义的节点：

```
specs/需求.md
specs/investment_schema.md
系统层级与模块名称必须来源于 output/建设方案.md
```

6. 图件仅用于表达既定技术结构，不得用于扩展系统能力。

7. 图件 JSON 文件命名规则：

```
文件名必须与 chart_type 一致
```

示例：

```
output/chart_spec/system_architecture.json
output/chart_spec/network_topology.json
output/chart_spec/data_flow.json
output/chart_spec/deployment_structure.json
```

8. 图件生成流程必须遵循以下顺序：

```
specs/需求.md
        ↓
specs/investment_schema.md
        ↓
output/建设方案.md
        ↓
图件 JSON
        ↓
HTML 图件
        ↓
PNG 图件
```

9. 禁止根据图件反推建设方案内容，
   图件仅作为 **建设方案结构表达结果**。

10. 节点命名必须使用设备或模块通用名称，
禁止包含品牌、型号或IP地址。
