你现在是一个系统架构工程师和前端图形引擎工程师。

当前项目：
proposal-writing-skill

项目中已经存在一个图件引擎：

chart/
├─ chart_schema.md
├─ layout_engine.js
├─ renderer.html
├─ icons.svg
└─ style.css

当前图件生成流程：

AI
↓
chart_data.json
↓
layout_engine.js
↓
renderer.html
↓
HTML / PNG

该架构必须保持不变。

AI只允许输出：

chart_data.json

AI不得输出：

* HTML
* SVG
* 坐标

所有布局必须由 layout_engine.js 负责。

---

当前系统已经具备：

* 分层布局（layer）
* 自动节点布局
* HTML渲染
* 图标节点
* 基本连线

但仍有两个关键差距，需要升级。

---

升级目标：

实现两个能力：

1）系统模块框（Group Layout）
2）智能连线（Link Routing）

要求：

在保持当前架构不变的情况下升级。

---

升级一：系统模块框（Group Layout）

当前结构：

layer
↓
node

升级为：

layer
↓
group
↓
node

chart_data.json 必须新增：

groups

例如：

{
"groups":[
{
"name":"视频监控系统",
"nodes":["camera","nvr"]
},
{
"name":"RFID定位系统",
"nodes":["rfid_reader","rfid_server"]
}
]
}

实现要求：

1）layout_engine.js

必须支持 group layout

需要新增：

groupRects

算法要求：

* 根据 group 内 node 的 bounding box
* 自动计算 group 边界
* group 外扩 padding
* group 内节点保持自动布局

效果：

┌──────────────┐
│ 视频监控系统 │
│  摄像机      │
│  NVR         │
└──────────────┘

group 样式：

* 半透明背景
* 边框
* group title

---

升级二：智能连线（Link Routing）

当前：

节点之间直接 SVG line

问题：

节点多时：

* 线条交叉
* 结构不清晰

升级为：

折线路由（orthogonal routing）

连接方式：

node
↓
垂直
↓
水平
↓
node

例如：

camera
│
└───── switch

实现要求：

renderer.html 新增：

link routing

逻辑：

1）优先垂直连接
2）避免节点重叠
3）自动计算折线点

SVG path 示例：

M x1 y1
L x1 midY
L x2 midY
L x2 y2

必须支持：

* 箭头
* 多条连线
* 层间连接

---
