综合系统建设类 architecture_scope.md
一、项目建设属性裁决

本项目属于综合系统建设类项目。

核心目标：

构建或优化基础设施资源能力

部署与承载应用系统

实现现场设备接入与系统集成

表达系统功能结构与界面逻辑

满足国产化与合规要求

本项目属于工程建设类项目，
不属于产品研发或业务创新项目。

二、本期建设范围（允许内容）
（一）基础设施建设

服务器建设或替换

存储资源规划

网络结构优化

资源冗余与高可用设计

（二）应用系统部署与结构设计

允许：

功能模块结构设计

系统逻辑结构表达

数据模型设计（基础级）

应用与数据库分层部署

运行环境适配

不得：

引入未立项新业务模块

重构既有业务逻辑

（三）现场设备接入与系统集成

允许：

摄像头接入

周界设备对接

门禁系统对接

RFID设备接入

触摸屏终端部署

与平台系统的数据交互设计

设备属于接入对象，而非独立创新系统。

（四）界面结构表达（受控）

允许：

页面结构原型图（enhanced）

功能页面布局图

页面流程示意图

不得：

营销化UI

真实产品界面

高保真视觉设计

未立项功能界面

UI表达仅用于结构说明。

三、排除范围

禁止：

AI算法研发

云架构建设

多技术路线比较

未来能力承诺

技术趋势论证

四、技术路线边界

必须服从 technical_baseline.md。

不得引入：

对象存储

云平台

异构数据库

未裁决技术模型

五、图件表达边界

必须生成：

总体架构图

网络拓扑图

部署结构图

设备接入结构图

如涉及功能表达：

可生成 UI 结构图（enhanced）

图件必须可由需求与 investment_schema 推导。

六、裁决优先级

如正文与本文件冲突，必须服从本文件。

七、规模与结构生成约束

本项目中：

所有服务器数量、设备数量、容量规模
必须来源于需求文件或投资清单。

不得基于“合理预估”自行扩展规模。

不得引入 investment_schema 和 需求中未出现的设备类型。

不得新增中间件或系统组件。

不得生成未裁决的架构层级。

本文件用于保障生成结构的稳定性与投资闭合一致性。
