一、文件定位

本文件为本项目图件资产生成与控制的唯一注册清单。

图件属于：

技术决策表达资产 + 结构表达资产

不属于自由内容生成资产。

二、图件分层体系

图件分为四类：

A类：架构决策类（强制）
B类：实施与流程类
C类：投标增强类
D类：UI表达类

A类图件分为：

A-现状类
→ 必须可由 specs/需求.md 推导

A-建设类
→ 必须可由 specs/investment_schema.md 推导


三、A类：架构决策类（强制生成）

属于技术路线表达资产。
必须服从 technical_baseline 与 architecture_scope。

A-01 系统总体架构图

触发语义：
建设方案章节中，
出现“总体框架”功能型子章节

输出：
output/chart/system_architecture.html

A-02 网络拓扑图

触发条件（满足任一）：

章节功能标签 = network_topology

章节标题包含以下功能型关键词之一：

网络拓扑

网络结构

链路结构

接入结构

investment_schema.md 中存在：

网络交换设备

多节点部署

跨机房互联

输出路径：

output/chart/network_topology.html

强制校验：


不得出现ip地址

不得出现未注册网络设备

必须服从 architecture_scope




A-03 数据流转图

触发条件（满足任一）：

章节功能标签 = data_flow

章节标题包含：

数据流转

数据处理流程

数据回传

数据路径

正文出现以下结构性描述：

数据采集 → 存储 → 访问

数据回传机制

数据处理链路

输出路径：

output/chart/data_flow.html

强制校验：

数据节点必须来自 investment_schema

不得新增缓存层

不得新增中间件

不得新增数据库节点

A-04 部署结构图

触发条件（满足任一）：

章节功能标签 = deployment_structure

章节标题包含：

部署结构

机房部署

服务器部署

物理部署

investment_schema.md 存在：

多服务器

多机柜

多机房

输出路径：

output/chart/deployment_structure.html

强制校验：

必须可由 investment_schema 推导

不得推导新增服务器

不得引入双中心

不得引入灾备结构

不得出现完整的ip地址

A-05 服务器角色划分图

触发条件（必须全部满足）：

investment_schema.md 中服务器节点数量 ≥ 3

存在不同功能角色（例如：

应用服务器

存储节点

管理节点）

architecture_scope 未禁止角色区分

输出路径：

output/chart/server_role_structure.html

强制校验：

不得新增未定义角色

不得拆分存储节点为控制/数据节点

不得引入数据库角色

不得出现完整的ip地址

A类强制规则

章节语义标签 + 配置文件校验 + 基线裁决

不得引入未定义节点

不得出现异构架构

不得出现对象存储

不得生成云架构模型

图件必须可由 investment_schema 推导


A-06 现网结构图


触发条件（满足全部）

章节功能标签 = current_architecture

章节语义包含以下任一功能型表达：

信息化现状

现网系统结构

现有系统架构

specs/需求.md 中存在系统结构事实描述

若 specs/需求.md 无结构事实：

不得生成图件。

输出路径
output/chart/current_architecture.html
事实来源（唯一合法来源）

specs/需求.md

禁止引用：

investment_schema.md

technical_baseline.md（不得改造）

正文优化性描述

强制校验规则

图件必须完全可由 specs/需求.md 推导

不得引入未在需求文件中出现的节点

不得删除需求文件中的节点

不得重命名系统组件

不得调整系统层级

不得新增逻辑分层

不得引入未来规划能力

架构范围校验（强制）

必须服从 architecture_scope


architecture_scope

即使为现网：

不得出现被禁止元素

不得出现云平台

不得出现灾备中心

表达原则

仅表达现网客观结构

不进行合理化改造

不进行架构优化

不进行技术评价

output/chart/ui_enhanced_xxx.html

D-03 页面流程交互图

D-04 系统界面示意图（投标可用）

触发：

D-05 现网系统截图（严格管理）

存放路径：

必须标注“现网界面示意”

七、UI生成强制限制

不得展示未来扩展能力

不得生成营销大屏

不得生成可运行代码

八、图件风格变量体系
技术图风格变量

CHART_STYLE = review | presentation | tender
默认：presentation

review（内部工程风）

定位：技术事实表达

视觉控制：

主色限定为黑 / 深灰

不使用大面积色块

仅允许少量状态色（≤ 2 种）

不允许结构强化条

不允许风险提示条

不允许强调底色分区

不允许视觉层级装饰

适用场景：

立项审批

技术备案

内部审查

目标效果：

像“技术记录文件”，
不是“表达增强材料”。

presentation（汇报增强风）

定位：管理理解表达

视觉控制：

允许主分区色（≤ 3 类）

允许浅底色块分区

允许状态色标识

允许轻度边框强化

不允许风险强化条

不允许对比式结构压强

不允许视觉焦点引导设计

适用场景：

领导汇报

专题汇报

结构说明

目标效果：

结构清晰、易读，
但不产生“投标表达感”。

tender（投标结构强化风）

定位：结构裁决表达

视觉控制：

允许明显分区对比

允许强调条（border-left / 分区线）

允许风险提示区块

允许状态色强化

允许模块结构边框强化

不允许渐变

不允许科技光效

不允许营销大屏

不允许品牌视觉风格

适用场景：

投标文件

专家评审

评分场景

目标效果：

专家在看结构逻辑，
而不是在看视觉设计。

九、图件生成纪律（统一规则）

所有正式图件必须先生成 HTML 源图

PNG 必须由 HTML 导出

分辨率 ≥ 1920×1080

图件生成不得触发章节重写

图件不得新增能力

图件不得脱离 investment_schema

十、图件冲突裁决优先级

若发生冲突，优先级如下：

1️⃣ architecture_scope
2️⃣ technical_baseline
3️⃣ investment_schema
4️⃣ chart_registry
5️⃣ 正文描述

十一、禁止行为

禁止：

新增未注册图件类型

输出实验性架构图

输出营销大屏

生成真实产品 UI

使用 ASCII 图替代正式图件

将 PNG 作为源图

十二、工程目标

目标是：

✔ 架构表达标准化
✔ UI表达边界化
✔ 投标得分工程化
✔ 图件生成自动化
✔ 风险控制制度化
