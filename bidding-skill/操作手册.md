投标专员操作手册

（基于 Bidding Skill 工程化规则）

一、手册适用范围

本手册适用于使用 Bidding Skill 投标文件工程化系统 进行政府采购、公安装备、信息化项目投标文件编制的投标专员。

系统结构与流程依据：

readme.md（工程流程说明）

readme

SKILL.md（行为规则与强制约束）

SKILL

第一部分：整体工作流程理解
标准流程（必须按顺序执行）
招标文件
   ↓
解析为 shared/（结构化权威输入）
   ↓
填写各公司 projects/X/specs/
   ↓
逐个生成 A → B → C
   ↓
交叉检查
   ↓
生成 Word 正式文件

⚠ 禁止跳步骤
⚠ 禁止直接写方案
⚠ 禁止未结构化就生成文件

第二部分：详细操作步骤
步骤一：解析招标文件（建立权威输入）
目标

将原始招标文件（PDF/Word）结构化为 shared/ 权威输入域。

操作方式（在 Claude 中执行）

输入固定指令：

按 bidding skill 的解析规则，对我提供的招标文件原文，只做信息提取，不允许总结、不允许改写、不允许解释，按 shared/ 结构输出。

必须生成：

shared/
├── 招标文件摘要.md
├── 评分标准.md
├── 技术要求.md
├── 商务要求.md
├── 资格要求.md
├── 废标条款.md
关键纪律

只提取原文

不优化语句

不改写

不补充行业常识

必须人工审核后确认

⚠ 未审核确认，不得进入生成阶段。

步骤二：填写投标主体信息（projects/X/specs/）

每个投标主体必须独立填写：

projects/A/specs/

包含：

文件	内容来源
公司资料.md	公司真实资料
报价清单.md	财务测算结果
偏离表.md	偏离声明
style_profile.md	风格差异化策略

⚠ 三家公司必须真实隔离
⚠ 不得复制粘贴

步骤三：生成投标文件（逐个主体）
原则：一次只处理一个主体

例如生成 A 公司：

按 bidding Skill，基于 shared/ + projects/A/specs/ 生成 A 的技术方案至 projects/A/output/

生成内容包括：

技术方案.md

商务方案.md

报价文件.md

资格文件.md

偏离表.md

废标风险检查.md

chart/

⚠ 完成 A 后必须新开对话生成 B
⚠ 禁止同一对话生成多个主体

第四部分：写作纪律（投标专员必须理解）

依据 SKILL.md 强制规则

SKILL

1. 技术方案规则

必须逐条响应技术要求

★参数必须正面响应

不得编造性能数据

不得引入未定义系统能力

不得删改招标章节编号

2. 商务方案规则

必须逐条响应商务要求

承诺必须与偏离表一致

不得超公司实际能力

3. 报价文件规则

报价唯一来源：报价清单.md

必须算术校验

不得超过控制价

禁止凭经验修改报价

4. 公司资料规则

不得编造业绩

资质必须真实

有效期覆盖投标截止日

第五部分：图件生成操作

当 specs 中已明确结构信息时，可生成：

系统架构图

网络拓扑图

部署结构图

数据流转图

图件规则：

必须HTML源文件

输出至 output/chart/

不得引入未定义元素

分辨率 ≥1920×1080

仅表达，不设计新结构

若结构不足：

必须生成：

output/图件输入缺口清单.md
第六部分：交叉检查（防串标）

三份完成后执行：

python cross_check.py

检查：

文本相似度 <40%

无公司信息泄露

报价无等差/等比/整数倍规律

图件配色区分

Word 元数据区分

⚠ 未通过不得交付

第七部分：生成 Word 文件

执行：

python projects/A/output/build_docx.py --all

分别为 A/B/C 生成。

禁止：

文件名含 A版/B版

作者信息相同

元数据未清洁

第十部分：执行纪律总结

shared/ 是唯一招标权威域

projects/X/specs/ 是唯一投标输入域

一次只生成一个主体

不得推理补充结构

不得编造公司信息

不得越权修改招标结构

一份合格的投标文件必须：

技术逐条响应

商务承诺一致

报价算术正确

无串标风险

图文一致

Word 格式规范
