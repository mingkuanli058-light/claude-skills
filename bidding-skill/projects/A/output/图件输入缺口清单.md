# 图件输入缺口清单

> 依据：SKILL.md 图件生成规则第二条、第四条
> 触发原因：技术方案涉及系统层级、组件构成、数据流转、接口关系，按规则须生成图件，但 specs/ 提供的结构化信息不足以支持合规生成，禁止基于常识或行业惯例补充。

---

## 一、应生成但无法生成的图件

### 1.1 系统总体架构图（优先级最高）

**触发依据**：技术方案涉及系统层级（感知层设备 / 应用软件层 / 数据层）、组件构成（26项软件模块、23类硬件设备）及外部对接关系。

**缺失信息**：

| 缺口项 | 说明 |
|-------|------|
| 系统分层定义 | specs 未定义系统的层级划分（感知层/平台层/应用层等） |
| 软件组件归属关系 | 26项软件功能模块的归属平台及层级关系未定义 |
| 硬件与软件绑定关系 | 哪些硬件接入哪个软件平台，specs 未说明 |
| 服务器与软件承载关系 | 平台管理软件、库管系统部署于哪台服务器，specs 未说明 |

---

### 1.2 数据流转图

**触发依据**：技术方案涉及数据流转（执法办案平台回传、分局数据上传、出入库预警数据对接）及数据存储（数据库、磁盘阵列）。

**缺失信息**：

| 缺口项 | 说明 |
|-------|------|
| 数据流向定义 | 各分局 → 管理中心 → 外部平台的完整数据流向，specs 未定义 |
| 接口协议类型 | 各对接接口的协议类型（API/数据库直连等），specs 未说明 |
| 数据存储层级 | 数据存储于磁盘阵列还是服务器本地，specs 未区分 |
| 外部系统接口端点 | 执法办案平台、政法系统、远程示证的接入端点，specs 未提供 |

---

### 1.3 部署结构图

**触发依据**：技术方案涉及系统部署位置（管理中心 + 各分局节点）、服务器和存储节点分布。

**缺失信息**：

| 缺口项 | 说明 |
|-------|------|
| 部署节点数量及位置 | 管理中心与各分局的物理分布，specs 未定义具体节点数 |
| 网络区域划分 | 公安网内部的区域隔离关系，specs 未说明 |
| 服务器与存储的部署关系 | 1台存储管理服务器 + 12台磁盘阵列的连接关系，specs 未定义 |

---

### 1.4 网络拓扑图

**触发依据**：技术方案涉及网络设备（防火墙1个）、设备接入（摄像机、门禁、RFID等），存在网络连接关系。

**缺失信息**：

| 缺口项 | 说明 |
|-------|------|
| 网络拓扑结构 | 设备接入网络的拓扑形态（星形/树形等），specs 未定义 |
| 网段划分 | 视频监控、门禁、RFID各设备所在网段，specs 未定义 |
| 防火墙策略与位置 | 防火墙在网络中的位置及策略，specs 未说明 |
| 分局接入方式 | 各分局通过何种链路接入管理中心，specs 未说明 |

## 二、需采购人补充的信息

| 缺口类别 | 需提供内容 | 优先级 |
|---------|---------|--------|
| 系统架构说明文档 | 系统分层结构、各组件归属平台 | 高 |
| 网络拓扑图（现有） | 采购人或原建设方提供的实际网络拓扑 | 高 |
| 部署说明 | 服务器/存储的实际部署位置与连接关系 | 高 |
| 数据流说明 | 分局→管理中心→外部系统的数据流向说明 | 中 |
| 接口文档 | 各外部系统对接的接口规范或现有对接方案 | 中 |

## 三、临时处置方案

在采购人补充上述信息前：

- 图件生成暂停，不以常识或行业惯例替代
- 技术方案正文以文字表格方式描述服务范围，不引入未确认的系统结构
- 待采购人提供上述文件后，按 SKILL.md 图件规则生成正式 HTML 源图件并输出至 output/chart/
